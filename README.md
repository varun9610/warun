# RTAF PEMS - คู่มือระบบและคู่มือการติดตั้ง
## Pilot Equipment Management System (ระบบบริหารจัดการอุปกรณ์และพัสดุสำหรับนักบิน กองทัพอากาศไทย)

ระบบ **RTAF PEMS** เป็นระบบเว็บแอปพลิเคชันที่มีความมั่นคงปลอดภัยสูง ได้รับการออกแบบขึ้นมาสำหรับกองทัพอากาศไทย เพื่อบริหารจัดการและควบคุมวงจรชีวิต (Lifecycle) ของสิ่งอุปโภคบริโภคจำเพาะของนักบิน (เช่น หมวกบิน Helmet, ชุดต้านแรงจี G-Suit และหน้ากากออกซิเจน Oxygen Mask) 

ระบบนี้บูรณาการเข้ากับระบบยืนยันตัวตนแบบหลายปัจจัยของ ทอ. (**RTAF Multi-Factor Authentication - MFA**) ผ่านรหัส OTP 6 หลัก เพื่อทดแทนระบบการล็อกอินแบบเก่าที่เป็นช่องโหว่ทางความปลอดภัยไซเบอร์

---

## 1. เทคโนโลยีที่เลือกใช้ (Technology Stack)

ระบบนี้พัฒนาขึ้นด้วยการเลือกสรรเฟรมเวิร์กและเทคโนโลยีที่ทันสมัยเพื่อเสถียรภาพ ประสิทธิภาพ และความปลอดภัยสูงสุด:

*   **ภาษาโปรแกรมหลัก (Language):** [TypeScript](https://www.typescriptlang.org/) (Type-safe ช่วยป้องกันข้อผิดพลาดของซอร์สโค้ดตั้งแต่ขั้นตอนการเขียน)
*   **เฟรมเวิร์กหน้าบ้าน (Frontend Framework):** [React 19](https://react.dev/) ร่วมกับ [Vite 6](https://vite.dev/) สำหรับการคอมไพล์และเสิร์ฟหน้าเว็บที่รวดเร็วเป็นพิเศษ
*   **เครื่องมือจัดแต่งหน้าตา (Styling CSS):** [Tailwind CSS v4](https://tailwindcss.com/) ร่วมกับระบบการกำหนดธีม (Theme Configuration) ที่ทันสมัย ให้ความละเอียดประณีต สบายตา มีคอนทราสต์ที่ถูกต้องตามมาตรฐานความเหลื่อมล้ำทางสายตาและสอดคล้องกับธีมกรมท่า-ทองของกองทัพ
*   **ระบบภาพเคลื่อนไหว (Animations):** [Motion](https://motion.dev/) (สืบทอดจาก `motion/react`) เพื่อสร้างการเปลี่ยนผ่านหน้าจออย่างลื่นไหล สมจริง และเป็นมืออาชีพ
*   **ระบบการจัดเก็บข้อมูล (Storage & Persistence):** บูรณาการระบบจัดเก็บฐานข้อมูล [Firebase Firestore](https://firebase.google.com/) สำหรับซิงโครไนซ์สถานะพัสดุ คำขอเบิก และบันทึกกิจกรรม (Audit Logs) แบบเรียลไทม์
*   **ชุดไอคอนและภาพสัญลักษณ์:** [Lucide React](https://lucide.dev/) และ Google Material Symbols เพื่อคงความสวยงามและแสดงผลทางทหารที่คมชัดในระดับเวกเตอร์

## 2. ขั้นตอนและกระบวนการทำงานของระบบ (System Walkflows)

ระบบแบ่งขั้นตอนการดำเนินงานออกเป็น 5 บทบาทหลัก (Roles) ที่ส่งข้อมูลสอดประสานกันดังนี้:

### ขั้นตอนที่ 1: การลงชื่อเข้าใช้งานอย่างปลอดภัยด้วย RTAF MFA (Authentication)
1.  ผู้ใช้ระบุ **ชื่อผู้ใช้งาน ทอ.** (RTAF Username / Email Prefix) เช่น `somchai_rtaf` (ระบบจะเชื่อมต่อแบบ `@rtaf.mi.th` เสมอ)
2.  ผู้ใช้คลิกขอรับรหัสผ่าน OTP เพื่อยืนยันตัวตนแบบ MFA ระบบจะจำลองส่งคำขอผ่านโพรโทคอลแบบ Secure ไปยัง Gateway `https://otp.rtaf.mi.th/api/v2/mfa/login`
3.  รหัสผ่าน OTP 6 หลักจะแสดงในหน้าต่างข้อความ SMS จำลองบนเว็บเพื่อให้ผู้ใช้คัดลองและป้อนข้อมูล
4.  ผู้ใช้กรอกรหัส 6 หลักเพื่อเข้าใช้งาน โดยมีหน้าต่าง **RTAF_AUTHENTICATION Protocol Monitor** แสดง Logs การทำงานจริงของระบบหลังบ้าน (เช่น การเรียกใช้ `BasicBackend` และการปฏิเสธ `PasswordBackend` เพื่อความปลอดภัย)
5.  เมื่อยืนยันถูกต้อง ระบบจะออกรหัสลับสิทธิ์การเข้าถึงแบบ **JWT Token** และพาผู้ใช้เข้าสู่หน้าแผงควบคุมตามระดับสิทธิ์ที่ได้รับ

### ขั้นตอนที่ 2: การยื่นคำขอเบิกของนักบิน (Pilot Workflow)
1.  **นักบิน (Pilot)** สามารถเข้าดูอุปกรณ์ที่ตนสวมใส่และอายุการใช้งานที่เหลืออยู่ได้ในแผงควบคุมส่วนตัว
2.  เมื่อต้องการเบิกพัสดุชิ้นใหม่ นักบินจะเข้าไปที่เมนู **"ยื่นคำขอเบิกพัสดุ"** 
3.  เลือกประเภทอุปกรณ์จำเพาะ (หมวกบิน Helmet, ชุด G-Suit, หรือหน้ากากออกซิเจน Oxygen Mask) ระบุจำนวน และเขียนระบุเหตุผลความจำเป็นในการเบิก (เช่น อุปกรณ์เดิมชำรุด หรือหมดอายุการใช้งาน)
4.  กดยื่นคำขอเบิกพัสดุ ระบบจะทำการบันทึกข้อมูลและอัปเดตสถานะเป็น **"รอดำเนินการ (Pending)"** ไปยังฐานข้อมูลทันที

### ขั้นตอนที่ 3: การตรวจสอบและอัปเดตสต๊อกโดยเจ้าหน้าที่พัสดุ (Supply Workflow)
1.  **เจ้าหน้าที่พัสดุ (Supply Officer)** จะได้รับแจ้งเตือนเมื่อมีคำขอเบิกพัสดุใหม่เข้ามาในระบบ
2.  เจ้าหน้าที่เข้าตรวจสอบระดับคลังสินค้า (Inventory Stock) โดยอ้างอิงรหัสพัสดุมาตรฐานองค์การนาโต (**NSN - National Stock Number**)
3.  เจ้าหน้าที่มีอำนาจคัดกรองคำขอ และเตรียมพัสดุเพื่อจัดส่ง หากพัสดุพร้อมจัดส่ง จะส่งคำขอที่มีความถูกต้องนี้ต่อไปยังผู้บังคับบัญชาเพื่อพิจารณาอนุมัติผ่านระบบ และอัปเดตสถานะของคำขอเบิก

### ขั้นตอนที่ 4: การลงนามอนุมัติโดยผู้บังคับบัญชา (Commander Workflow)
1.  **ผู้อนุมัติ (Commander)** เข้าใช้งานระบบเพื่อตรวจสอบเอกสารและพัสดุที่มีมูลค่าสูงซึ่งได้รับการคัดกรองจากแผนกพัสดุแล้ว
2.  ผู้บังคับบัญชาสามารถเลือก **"อนุมัติ (Approve)"** หรือ **"ปฏิเสธการอนุมัติ (Reject)"** พร้อมระบุข้อคิดเห็น (Comments) แนบไปในเอกสารคำขอเบิกได้
3.  เมื่ออนุมัติเสร็จสิ้น ระบบจะเปลี่ยนสถานะเป็น **"กำลังจัดส่ง (In Transit)"** หรือหักยอดคงเหลือในระบบพัสดุจริงโดยอัตโนมัติ

### ขั้นตอนที่ 5: การสรุปรายงานเชิงวิเคราะห์และการตรวจสอบระบบ (Executive & Admin Workflows)
1.  **ผู้บริหาร (Executive)** สามารถดูแนวโน้มการเบิกพัสดุ มูลค่างบประมาณที่ใช้ และอัตราความชำรุดของอุปกรณ์ผ่านระบบ Visual Dashboard เพื่อวิเคราะห์ประเมินงบประมาณการสั่งซื้อล่วงหน้าของกองทัพ
2.  **ผู้ดูแลระบบ (System Admin)** เข้าตรวจสอบสถานะการทำงานของเซิร์ฟเวอร์ และตรวจสอบ **บันทึกประวัติเหตุการณ์การใช้งาน (Audit Logs)** เพื่อเช็กประวัติความมั่นคงปลอดภัยว่ามีใครขอเบิก แก้ไข หรือยืนยันตัวตนสำเร็จ/ล้มเหลว ณ เวลาใดบ้าง ป้องกันการทุจริตและการบุกรุกระบบอย่างเข้มงวด

## 3. คู่มือการติดตั้งและเริ่มใช้งาน (Installation Guide)

โปรดทำตามขั้นตอนต่อไปนี้เพื่อนำระบบไปติดตั้งและใช้งานในเครื่องคอมพิวเตอร์ของท่าน:

### ความต้องการขั้นต่ำของระบบ (Prerequisites)
*   **Node.js:** เวอร์ชัน 18.x หรือใหม่กว่า (แนะนำ LTS)
*   **npm:** เวอร์ชัน 9.x หรือใหม่กว่า

### ขั้นตอนการติดตั้งอย่างเป็นขั้นตอน (Step-by-Step Installation)

#### ขั้นตอนที่ 1: การดาวน์โหลดซอร์สโค้ดของโปรเจกต์
ดาวน์โหลดหรือทำการโคลนซอร์สโค้ดจากที่เก็บข้อมูล (Git Repository) ลงในโฟลเดอร์ที่ต้องการบนเครื่องคอมพิวเตอร์ของท่าน

#### ขั้นตอนที่ 2: เปิดโปรแกรม Command Prompt หรือ Terminal
ใช้คำสั่งเปลี่ยนโฟลเดอร์การทำงานไปยังไดเรกทอรีรากของโปรเจกต์:
```bash
cd rtaf-pems-system
```

#### ขั้นตอนที่ 3: ติดตั้งแพ็กเกจและ Dependencies ทั้งหมด
ระบบจะทำการดาวน์โหลดและติดตั้งโมดูลที่ระบุไว้ในไฟล์ `package.json` เช่น React, Tailwind CSS, Motion และโมดูลเสริมอื่นๆ:
```bash
npm install
```

#### ขั้นตอนที่ 4: การกำหนดค่าตัวแปรสภาพแวดล้อม (Environment Variables)
1.  คัดลอกไฟล์ต้นแบบ `.env.example` และเปลี่ยนชื่อเป็น `.env` บนไดเรกทอรีราก:
    ```bash
    cp .env.example .env
    ```
2.  เปิดไฟล์ `.env` และกำหนดค่าความลับของระบบ เช่น คีย์เชื่อมฐานข้อมูล Firebase และ API คีย์ของ RTAF OTP Server:
    ```env
    # ตัวอย่างไฟล์ .env
    VITE_FIREBASE_API_KEY=your_firebase_api_key
    VITE_FIREBASE_AUTH_DOMAIN=your_auth_domain
    VITE_FIREBASE_PROJECT_ID=your_project_id
    RTAF_MFA_GATEWAY_URL=https://otp.rtaf.mi.th/api/v2/mfa/login
    ```

#### ขั้นตอนที่ 5: รันระบบในโหมดนักพัฒนา (Run Dev Server)
เปิดใช้งานเซิร์ฟเวอร์จำลองบนเครื่องคอมพิวเตอร์ของคุณเพื่อการทดสอบและการพัฒนาซอฟต์แวร์:
```bash
npm run dev
```
หลังรันสำเร็จ เซิร์ฟเวอร์จะเปิดเสิร์ฟหน้าเว็บที่แอดเดรส:
`http://localhost:3000` (หรือ URL ที่ระบุบนหน้าจอ Command)

#### ขั้นตอนที่ 6: การทดสอบความถูกต้องและจัดหน้าตาของซอร์สโค้ด (Linting)
คุณสามารถสั่งตรวจสอบข้อผิดพลาดด้านหลักไวยากรณ์และความสอดคล้องของประเภทข้อมูล (Type checking) ผ่านคำสั่ง:
```bash
npm run lint
```

#### ขั้นตอนที่ 7: การสร้างแพ็กเกจสำหรับนำไปติดตั้งใช้งานจริง (Build for Production)
เมื่อพัฒนาระบบเสร็จสมบูรณ์และต้องการนำไป Deploy บนโฮสต์ของกองทัพอากาศ ให้รันคำสั่งสำหรับบีบอัดและคอมไพล์โค้ดให้อยู่ในรูปไฟล์ HTML/JS คุณภาพสูง:
```bash
npm run build
```
ผลลัพธ์ที่ได้จะอยู่ในโฟลเดอร์ `dist/` ซึ่งเป็นไฟล์ Static เว็บแอปพลิเคชันที่พร้อมนำไปอัปโหลดขึ้นเครื่องแม่ข่ายจริง

## 4. โครงสร้างไฟล์ที่สำคัญในระบบ (Project Directory Structure)

*   `/src/App.tsx` - ไฟล์ควบคุมหลักสำหรับสลับหน้าจอ (Routing) และจัดการเซสชันของผู้ใช้ที่ลงชื่อเข้าใช้
*   `/src/components/LoginView.tsx` - หน้าจอการลงชื่อเข้าใช้ระบบ (RTAF OTP / MFA) พร้อม **Protocol Monitor แสดง Logs** การเรียก API จริงไปยัง otp.rtaf.mi.th
*   `/src/components/PilotHomeView.tsx` - แผงควบคุมและประวัติส่วนตัวของนักบิน
*   `/src/components/PilotRequestView.tsx` - แบบฟอร์มการยื่นคำขอเบิกพัสดุจำเพาะของนักบิน
*   `/src/components/SupplyHomeView.tsx` - แผงควบคุมระดับคลังสิ่งของ รหัส NSN และการตรวจรับพัสดุเข้าคลัง
*   `/src/components/CommanderHomeView.tsx` - ระบบการพิจารณา ตรวจสอบราคา และลงนามอนุมัติคำขอเบิกโดยผู้บังคับบัญชา
*   `/src/components/ExecutiveHomeView.tsx` - แผงรายงานสถิติเชิงกราฟและการประเมินงบประมาณการใช้อุปกรณ์
*   `/src/components/AdminHomeView.tsx` - แผงควบคุมระบบ บันทึกเหตุการณ์ (Audit Logs) และการกำหนดผู้ใช้
*   `/src/lib/firebase.ts` - ส่วนเชื่อมต่อและการส่งข้อมูลแบบเรียลไทม์ผ่าน Firebase Database
*   `/src/types.ts` - ส่วนกำหนดคลาสข้อมูล แบบจำลอง และประเภทข้อมูลทั้งหมดของนักบินและพัสดุ
*   `/package.json` - ส่วนกำหนดเวอร์ชันของระบบ สคริปต์ และแพ็กเกจเสริมทั้งหมดที่ใช้
